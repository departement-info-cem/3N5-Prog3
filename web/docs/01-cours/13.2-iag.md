## Exercice Modeles.md

Le plugin copilot dans Intellij te permet de choisir entre différents modèles d'IAG (Claude, Gemini, GPT etc.).

On va reproduire l'expérience avec les modèles suivants:
- GPT-4.1  dans un projet appelé **AppliGPT**
- Claude Haiku 4.5 dans un projet appelé **AppliClaude**
- Gemini Pro 2.5 dans un projet appelé **AppliGemini**

Pour chaque modèle, fais les étapes suivantes:
1. Créer un projet de départ Android Studio avec le bon nom dans ton repo
2. Ouvre le mode Agent de Copilot et choisis le modèle approprié.
3. On va commencer avec le prompt suivant :
```text
crée une interface compose jetpack avec une topbar contenant le titre **Accueil**, 
un menu déclenché par le hamburger qui contient une zone en haut indiquant 
l'utilisateur connecté puis les options Accueil avec une icone de maison et 
profile avec une icone de profil. Dans le topBar je veux aussi une action avec 
l'icone de partage qui pour l'instant déclenche juste un println qui dit coucouc 
dans logcat, je veux aussi un floating action button**
```
4. Valider le bon fonctionnement de l'appli. Si ça marche pas le noter dans un commentaire
5. On va essayer un prompt beaucoup plus vague:
```text
Crée un écran de profil et traduis l'interface au complet
```
6. Valider le bon fonctionnement de l'appli. Si ça marche pas le noter dans un commentaire
7. Finalement on va utiliser un gros prompt très vague
```text
Gère la navigation entre les écrans existant, ajoute une liste de tâches à faire dans
 l'écran d'accueil plus tout ce qu'il faut pour pouvoir créer modifier une tâche qui
  est stockée en BD, plus des comptes avec inscription connexion et déconnexion aussi
  je voudrais un écran de chargement avec un logo généré qui montre que l'appli gère des TODO listes.
```

As-tu un favori? Pourquoi? Écris tout ça dans un fichier **Modeles.md** à la racine de ton repo.
- as-tu le contrôle précis sur le code produit?
- quel pourcentage du code tu comprends? 10? 20? 50? 80? 100?
- quel modèle t'a semblé le plus performant?
